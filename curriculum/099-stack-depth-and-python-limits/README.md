# Day 99 — The case of the function that worked yesterday

A teammate Slacks you. *"My JSON walker crashed in production. Worked
fine on my laptop. Same input."* Stack trace:

```text
RecursionError: maximum recursion depth exceeded
```

The function:

```python
def deep_keys(obj, prefix=""):
    if isinstance(obj, dict):
        for k, v in obj.items():
            yield from deep_keys(v, f"{prefix}.{k}")
    elif isinstance(obj, list):
        for i, v in enumerate(obj):
            yield from deep_keys(v, f"{prefix}[{i}]")
    else:
        yield prefix
```

You investigate. Three suspects.

## Suspect 1: missing base case

Your first instinct. But there's a base case — the `else` branch that
yields `prefix`. Crossed off the list.

## Suspect 2: it's actually deep

You ask for the input. It's a config tree generated by a templating
system, with a chain of nested dicts ~1,200 deep. Python's default
`sys.getrecursionlimit()` is **1000**. Each recursive call into a
nested dict pushes a frame; at 1001 you crash.

That's the bug. Confirmed:

```python
>>> import sys
>>> sys.getrecursionlimit()
1000
```

## Suspect 3: not actually a bug — Python design

The recursion limit isn't arbitrary. Each Python frame uses ~1 KB of
C stack. The OS gives each thread a fixed stack (often 8 MB). Recurse
too deep and you don't just hit a Python limit — you'd blow the C
stack and segfault. The limit is a guardrail, not a wall.

## Three fixes, in order of preference

**Fix 1: rewrite as iteration with an explicit stack.**
For traversals like this, replace recursion with a `to_visit` list
you `.pop()` from. No Python stack pressure at all.

```python
def deep_keys(obj):
    todo = [(obj, "")]
    while todo:
        cur, prefix = todo.pop()
        if isinstance(cur, dict):
            for k, v in cur.items():
                todo.append((v, f"{prefix}.{k}"))
        elif isinstance(cur, list):
            for i, v in enumerate(cur):
                todo.append((v, f"{prefix}[{i}]"))
        else:
            yield prefix
```

This handles arbitrary depth — your stack lives on the heap now.

**Fix 2: raise the limit, narrowly.**
If rewriting is too painful, `sys.setrecursionlimit(10_000)` will
work. But don't crank it without thinking — at some depth you stop
hitting the Python limit and start crashing the interpreter.

**Fix 3: check `sys.getrecursionlimit()` in your tests.**
A test like `assert process(deep_input) == expected` catches this on
CI before production. Always.

## The takeaway

Recursion isn't free in Python. For unbounded-depth data — JSON from
external systems, file trees, comment threads — assume the worst and
either bound your input or use the explicit-stack pattern.
