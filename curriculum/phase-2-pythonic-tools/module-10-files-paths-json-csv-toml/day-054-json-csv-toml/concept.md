# Day 54 — A code tour of `json`, `csv`, `tomllib`

Three formats. Three modules in the standard library. Same shape:
load → manipulate as Python data → dump.

```python
import json, csv, tomllib
from pathlib import Path

# JSON
data = json.loads(Path("config.json").read_text())
Path("out.json").write_text(json.dumps(data, indent=2))

# CSV (file-friendly, line-by-line)
with open("users.csv") as f:
    reader = csv.DictReader(f)
    users = [row for row in reader]

# TOML (read-only in stdlib)
config = tomllib.loads(Path("pyproject.toml").read_text())
```

Walk through each.

## `json` — for API/data interchange

`json.loads(text)` and `json.dumps(obj)` are the workhorse pair.
There are also file variants (`json.load(f)` / `json.dump(obj, f)`),
but `Path.read_text` + `json.loads` is more readable and works
identically.

The Python ↔ JSON mapping is exactly what you'd expect:

| Python | JSON |
|---|---|
| `dict` | object `{...}` |
| `list`, `tuple` | array `[...]` |
| `str` | string |
| `int`, `float` | number |
| `True`, `False` | `true`, `false` |
| `None` | `null` |

Anything else (a `Path`, a custom class, a `datetime`) raises
`TypeError`. For dataclasses, the easiest pattern is
`json.dumps(dataclasses.asdict(obj))`.

Useful kwargs:

- `indent=2` — pretty-print
- `sort_keys=True` — stable diffs in version control
- `default=str` — fallback that stringifies anything weird

## `csv` — for tabular data

CSV looks simple and isn't. Quoting, newlines inside quoted fields,
encoding — the `csv` module handles all of these, so you should never
hand-parse CSV.

Two sane shapes:

```python
# DictReader: each row is a dict keyed by header
import csv
with open("users.csv") as f:
    for row in csv.DictReader(f):
        print(row["name"], row["email"])

# Writer: each row is a list (or a dict + DictWriter)
with open("out.csv", "w", newline="") as f:
    w = csv.writer(f)
    w.writerow(["name", "email"])
    w.writerows([("Mada", "m@x"), ("Alex", "a@x")])
```

The `newline=""` on writes is **important** — without it, you get
double-newlines on Windows. Always pass it.

## `tomllib` — for human-edited config

TOML is a small, opinionated config format used by `pyproject.toml`
and many tools. Python 3.11+ ships `tomllib` for *reading* TOML.
Writing TOML isn't in stdlib (use `tomli-w` or `tomlkit`); for most
projects you don't need to write TOML programmatically.

```python
import tomllib
config = tomllib.loads(Path("pyproject.toml").read_text())
config["project"]["name"]   # 'swe-skills'
```

`tomllib.loads` returns a plain dict — same as `json.loads`. Same
shape, different on-disk format.

## Pick the format

- **Internal app data, structured, generated by code** → JSON.
- **Tabular, exported to a spreadsheet, exchanged with non-coders** → CSV.
- **Human-edited config files for tools** → TOML.

When in doubt, JSON. It's universal and lossless for everything
JSON-mappable.
